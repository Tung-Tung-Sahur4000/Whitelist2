What it can & cannot do

✅ It CAN

  • Log in with the token you put in config.yml
  • Receive its slash commands: /whitelist, /link, /unlink
  • Receive DMs — but only acts on messages that are exactly the code digits; all else ignored
  • Read a member’s roles (only to check the whitelist role)
  • Add / remove names in WhitelistedPlayers.yml

❌ It CANNOT

  • Read your server chat — it never requests the MESSAGE_CONTENT intent
  • Kick / ban members, change roles, delete messages, manage the server
  • Run arbitrary Minecraft commands or touch anything but the whitelist
  • Send the token anywhere — it only logs in with it
  • Ping @everyone / mention anyone (replies = mentions off)

Gateway intents it requests — verifiable in the Discord developer portal

DIRECT_MESSAGES — alwaysto receive code DMs (DM content needs no privileged intent)
GUILD_MEMBERS — only if role-sync is onto see who gains/loses the whitelist role (privileged)
MESSAGE_CONTENT — never→ this is why it cannot read your server chat
